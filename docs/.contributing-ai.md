# Contributing Guide for AI Agents

This document explains how AI should contribute code to the Meeovi repository.

---

## 1. Workflow

1. Read `.meeovi-architecture.md`.
2. Read `.ai-guidelines.md`.
3. Read `.invariants.md`.
4. Read relevant interfaces in `/contracts`.
5. Read existing patterns in the codebase.
6. Generate code that fits the architecture.
7. Ensure tests pass.
8. Produce a clear summary of changes.

---

## 2. Required Behaviors

AI must:
- Follow architectural rules.
- Follow coding standards.
- Follow security invariants.
- Follow existing patterns.
- Keep changes minimal and scoped.
- Explain reasoning clearly.

---

## 3. Pull Request Requirements

AI-generated PRs must include:
- Summary of changes.
- Files modified.
- Reason for changes.
- Confirmation that:
  - tests pass
  - architecture is respected
  - no public contracts were changed

---

## 4. Allowed Actions

AI may:
- Implement new adapters.
- Implement new capability methods.
- Add new composables.
- Add new tests.
- Refactor internal code (non-public).
- Improve documentation.

---

## 5. Forbidden Actions

AI may not:
- Change public interfaces.
- Change architecture files.
- Introduce new patterns.
- Add business logic to adapters.
- Modify tests without instruction.
- Add unvalidated input paths.

---

## 6. Escalation

If a task requires architectural changes, AI must:
- Stop.
- Notify the human architect.
- Provide options and tradeoffs.

---

AI is a contributor, not an architect.  
All architectural decisions require human approval.
