# Feature Packs

Feature Packs extend the M Framework itself with a bulk of features at a time.

Use feature packs when you need to:

- Add reusable framework capabilities that can be shared across apps and layers.
- Expose composables, utilities, runtime hooks, and extension points.
- Package cross-project features as portable building blocks.

Examples of a feature pack would be:

 - Add ecommerce functionality to an app creating a de facto ecommerce platform or CMS.
 
In short: feature packs are framework extensions.
